# ランタイムアセットとマップ生成

## 常駐データ

ゲームプレイ用BGとスプライトは、ビルド時にメタデータと画素領域へ分離します。頻繁に参照する
BG画素、スプライト記述表、スプライト画素は`.data.pixel_twins_assets`へ配置し、RP2350起動時に
FlashのロードイメージからSRAMへコピーします。BG論理参照表、パレット原本、タイトル素材は
Flashから直接参照します。

| データ | 容量 | 配置 |
| --- | ---: | --- |
| BG画素 | 125,952 bytes | SRAM |
| スプライト記述表・画素 | 173,123 bytes | SRAM |
| フォントグリフ | 1,710 bytes | SRAM |
| BG論理参照表 | 164 bytes | Flash |
| gameplayパレット原本 | 768 bytes | Flash |
| タイトル一枚絵 | 19,200 bytes | Flash |
| タイトルロゴ | 2,020 bytes | Flash |
| タイトルパレット原本 | 768 bytes | Flash |
| 100×100タイルマップ | 10,000 bytes | RAM |
| 4-bit地形ワーク | 5,000 bytes | マップ生成時RAM |

地形ワークはゲーム開始時の生成にだけ必要です。生成後は別用途へ再利用できます。

最新の詳細な容量は[生成メモリレポート](../assets/converted/memory_layout.md)に出力します。

## マップ生成

`world::MapGenerator::generate()`はヒープ確保を行わず、呼び出し元から渡された`TerrainWorkspace`と
`WorldMap`だけを使用します。同じ32-bitシードからは同じタイルマップを生成します。

生成順は次のとおりです。

1. 島形状と海岸のwater、sand、grassを生成
2. 中央と3つの封印石を結ぶdirt、roadを生成
3. 中央広場と封印石広場を生成
4. 地形の隣接状態から16方向境界マスクを決定
5. BGパックの論理参照表から0〜122の物理パターン番号へ変換
6. 衝突属性を上位1bitへ格納
7. 装飾と封印石の焼き込みパターンを配置

最終タイル値の下位7bitはBGパターン番号、上位1bitは衝突属性です。描画には
`tileIndexMask = 0x7f`を使用します。

## 描画とアニメーション

`WorldMap::draw()`は生成済みタイルマップとFlash上のパターンを`drawBackground()`へ直接渡します。
描画フレーム内で境界判定やパターン検索は行いません。

スプライトは生成された`SpriteAssetId`で選択します。`GameAssets::makeLoopingSprite()`は列方向を
時間フレームとして自動的に剰余処理し、行方向を向きとして指定します。透明外周のtrimオフセットは
スプライトパックから自動適用されます。

タイトルは`TitleAssets`が専用パレット、一枚絵、trim済みロゴを管理します。一枚絵は
`drawScreen()`でFlashから直接コピーし、ロゴは通常のスプライト描画へ渡せます。
